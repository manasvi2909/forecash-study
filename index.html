03 / Project Scope

The Team & Role

ForeCash is a Level 0 functional prototype — all screens are fully interactive and navigable, transactions are logged manually by the user, and ML forecasts run on the data entered within the session. It is not connected to any external bank or financial account. The prototype supports the complete core user journey: onboarding → transaction logging → cash flow forecasting → AI assistant → tax estimation, with real computed outputs based on user-entered figures. It is demonstration-ready and has been validated with 5 usability test participants.

11 / ML Architecture

How ForeCash thinks.

ForeCash doesn't just display numbers — it learns from them. Here's the model powering the forecast engine.

Forecasting Models

A hybrid of ARIMA (AutoRegressive Integrated Moving Average) for trend detection and a lightweight LSTM neural network for capturing non-linear spending patterns. The ensemble model adapts as more transaction history accumulates.

ARIMA • LSTM • Ensemble

Data Pipeline

Transactions are entered manually by the user — amount, category, date, and recurrence. Each entry is normalised and classified using a fine-tuned categorisation model, then stored as time-series data. Outliers are flagged and excluded from forecast windows.

Manual Entry • Normalise • Categorise • Store

Prediction Logic

The forecast engine generates a rolling 90-day cash flow projection, updated nightly. It factors recurring transactions, seasonal patterns, user-defined goals, and income variability to surface confidence intervals rather than single-point predictions.

90-day horizon • Confidence bands • Live updates

Security

All financial data is encrypted at rest (AES-256) and in transit (TLS 1.3). No raw credentials are stored.

Transactions sync every 4 hours; forecast recalculates nightly at 2am with the latest data window.

Scalability

Microservices architecture. The ML inference layer scales independently from the transaction API and UI serving layer.

Key Findings & Iterations

Fixed
Onboarding felt too long

4 of 5 users felt uncertain midway through setup. Solution: added a progress indicator and reduced steps from 7 to 5.

Fixed
Forecast graph lacked context

Users couldn't interpret the confidence bands. Solution: added an inline tooltip explaining the range in plain language.

Fixed
FinPilot AI entry point was hidden

3 of 5 users didn't discover FinPilot within 2 minutes. Solution: added a persistent floating action prompt on the dashboard.

Iterated
Tax section naming confusion

“Account Balance” label was misread as a bank balance view. Renamed to “Tax & Savings” with a clearer icon.

Noted
Users wanted export to CSV

All 5 users asked for data export. Logged as a high-priority feature for v1.1.

16 / Retrospective

Reflections & Learnings

Designing ForeCash highlighted the critical importance of clarity in financial interfaces. When users deal with their future financial stability, ambiguity induces anxiety.

By shifting the core interaction from tracking to forecasting, historical data became secondary context to predictive graphs. This entirely changed the data hierarchy.

The biggest architectural challenge was making complex predictive data visually intuitive without requiring a finance degree—emphasizing the need for minimalist charts and conversational AI translation.
